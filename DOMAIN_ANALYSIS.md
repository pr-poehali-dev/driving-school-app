# Анализ предметной области: Автошкола "АвтоПрофи"

## 1. Описание предметной области

**АвтоПрофи** — это автошкола, предоставляющая услуги обучения вождению транспортных средств различных категорий. Компания работает на рынке 15 лет и имеет более 5000 выпускников, 96% из которых сдают экзамены с первого раза.

### Основные направления деятельности:

1. **Обучение вождению** по различным категориям водительских удостоверений (A, B, C)
2. **Теоретическая подготовка** — изучение ПДД, правил безопасности, устройства автомобиля
3. **Практические занятия** — вождение с инструктором на учебных автомобилях
4. **Подготовка к экзаменам** в ГИБДД
5. **Пробные занятия** — бесплатное первое занятие для знакомства с процессом обучения

---

## 2. Основные бизнес-процессы

### 2.1 Процесс записи на курс

1. **Клиент выбирает курс** на сайте (категория A, B, C или пробное занятие)
2. **Заполняет заявку** с контактными данными (ФИО, телефон, email, комментарий)
3. **Заявка поступает** в систему со статусом "new"
4. **Менеджер обрабатывает заявку**:
   - Связывается с клиентом
   - Уточняет детали
   - Назначает время занятий
   - Меняет статус на "contacted" или "enrolled"
5. **Клиент начинает обучение** или проходит пробное занятие

### 2.2 Процесс обучения

1. **Теоретическая часть**:
   - Занятия в классе (130 часов для категории B)
   - Изучение ПДД
   - Устройство автомобиля
   - Оказание первой помощи

2. **Практическая часть**:
   - Вождение на автодроме (18-72 часа в зависимости от категории)
   - Вождение в городе
   - Работа с инструктором

3. **Подготовка к экзамену**:
   - Внутренний экзамен в автошколе
   - Сопровождение в ГИБДД
   - Сдача государственного экзамена

### 2.3 Управление инструкторами

1. **Найм инструкторов** с опытом от 5 лет
2. **Назначение на группы** и индивидуальные занятия
3. **Контроль качества** обучения (рейтинг инструкторов)
4. **Специализация** по категориям транспорта

---

## 3. Роли пользователей

### 3.1 Клиент (потенциальный ученик)
- Просматривает курсы
- Оставляет заявки на обучение
- Записывается на пробное занятие
- Изучает информацию об инструкторах

### 3.2 Менеджер / Администратор
- Обрабатывает заявки клиентов
- Просматривает статистику заявок
- Управляет курсами (создание, редактирование, удаление)
- Управляет инструкторами
- Меняет статусы заявок (new → contacted → enrolled → completed → cancelled)

### 3.3 Инструктор
- Проводит практические занятия
- Оценивает прогресс учеников
- Подготавливает к экзаменам

---

## 4. Ключевые сущности

### 4.1 Курс (Course)
**Описание**: Учебная программа по определенной категории водительского удостоверения

**Атрибуты**:
- ID курса
- Название курса (например, "Категория B")
- Категория (A, B, C)
- Описание программы
- Длительность обучения
- Стоимость (₽)
- Список преимуществ (features)
- Изображение
- Дата создания

**Примеры**:
- Категория B (легковой автомобиль) — 35,000₽, 3 месяца
- Категория A (мотоцикл) — 28,000₽, 2 месяца
- Категория C (грузовик) — 45,000₽, 4 месяца

### 4.2 Заявка на обучение (Enrollment)
**Описание**: Запрос клиента на запись на курс или пробное занятие

**Атрибуты**:
- ID заявки
- ФИО клиента
- Телефон (обязательно)
- Email (опционально)
- ID выбранного курса
- Комментарий/пожелания
- Статус заявки (new, contacted, enrolled, completed, cancelled)
- Дата создания

**Жизненный цикл статусов**:
```
new → contacted → enrolled → completed
  ↘                            ↓
    ──────────────→ cancelled
```

### 4.3 Инструктор (Instructor)
**Описание**: Преподаватель вождения

**Атрибуты**:
- ID инструктора
- ФИО
- Специализация (категории, которые преподает)
- Опыт работы (лет)
- Рейтинг (от 1.0 до 5.0)
- Биография
- Фото
- Дата добавления

**Примеры**:
- Петров Иван — категория B, 15 лет опыта, рейтинг 4.9
- Сидорова Мария — категории A и B, 8 лет опыта, рейтинг 4.8

---

## 5. Связи между сущностями

### Enrollment → Course (многие к одному)
- Одна заявка связана с одним курсом
- Один курс может иметь множество заявок
- Связь: `enrollments.course_id → courses.id`

**Бизнес-логика**:
- При записи клиент выбирает конкретный курс
- Заявка может быть на пробное занятие (course_id = 0 или NULL)
- Менеджер видит, какие курсы пользуются спросом

---

## 6. Бизнес-правила

### 6.1 Курсы
- Стоимость курса фиксирована при создании
- Каждый курс имеет уникальную категорию (A, B, C, и т.д.)
- Длительность обучения зависит от категории
- Количество часов теории и практики определяется требованиями ГИБДД

### 6.2 Заявки
- Телефон — обязательное поле для связи
- Email — опциональное поле
- Новая заявка всегда создается со статусом "new"
- Статус может быть изменен только администратором
- Заявка на пробное занятие бесплатна (цена = 0₽)

### 6.3 Инструкторы
- Минимальный опыт работы — 5 лет
- Рейтинг рассчитывается на основе отзывов учеников
- Каждый инструктор имеет специализацию по категориям

### 6.4 Пробное занятие
- Длительность: 45 минут
- Бесплатно для новых клиентов
- Включает знакомство с инструктором и первый урок вождения
- Место проведения выбирается клиентом

## 7. Требования к системе

### 7.1 Функциональные требования

**Для клиентов:**
- Просмотр каталога курсов с ценами и описаниями
- Запись на курс через форму
- Запись на бесплатное пробное занятие
- Просмотр информации об инструкторах

**Для администраторов:**
- Просмотр всех заявок в едином интерфейсе
- Фильтрация заявок по статусу и дате
- Изменение статуса заявки (new → contacted → enrolled, и т.д.)
- Управление курсами (CRUD операции)
- Управление инструкторами (CRUD операции)
- Просмотр статистики (количество заявок, популярные курсы)

### 7.2 Нефункциональные требования

**Производительность:**
- Быстрая загрузка главной страницы (< 2 секунд)
- Мгновенное открытие формы записи

**Удобство использования:**
- Адаптивный дизайн для мобильных устройств
- Простая и понятная форма записи
- Минимальное количество обязательных полей

**Безопасность:**
- Защита персональных данных клиентов
- Доступ к админ-панели только для авторизованных пользователей
- Защита от SQL-инъекций и XSS-атак

**Надежность:**
- Сохранение заявок в базе данных
- Резервное копирование данных
- Уведомления при получении новой заявки

## 8. Показатели эффективности (KPI)

### Для бизнеса:
- **Конверсия заявок** → число записавшихся / общее число заявок
- **Средний чек** — средняя стоимость курса
- **Популярность курсов** — количество заявок по категориям
- **Процент сдачи с первого раза** — 96% (целевой показатель)

### Для маркетинга:
- Количество новых заявок в день/неделю/месяц
- Процент заявок на пробное занятие
- География клиентов (если добавить поле)

### Для качества обучения:
- Рейтинг инструкторов (> 4.5 — отлично)
- Количество учеников на одного инструктора
- Время обработки заявки (< 1 рабочий день)

3. **Календарь занятий** для записи на конкретное время

   - Прогресс обучения
   - Оплата курса онлайн
   - История платежей

2. **Интеграция с ГИБДД**:
   - Запись на экзамен онлайн
   - Проверка статуса готовности документов

3. **Мобильное приложение** для учеников

### Долгосрочные (12+ месяцев):
1. **Онлайн-теория** — видеоуроки, тесты ПДД
2. **Система лояльности** — скидки для повторных учеников
3. **Франчайзинг** — открытие филиалов в других городах
4. **CRM-система** для управления клиентами

## 11. Риски и ограничения

### Бизнес-риски:
- Сезонность спроса (весна-лето — пик, зима — спад)
- Конкуренция с другими автошколами
- Изменения в законодательстве (требования ГИБДД)

### Технические риски:
- Потеря данных клиентов (решение: регулярные бэкапы)
- Недоступность сайта (решение: облачный хостинг)
- Утечка персональных данных (решение: шифрование, HTTPS)

### Операционные ограничения:
- Ограниченное количество инструкторов (max 10-15 человек)
- Ограниченное количество учебных автомобилей
- График работы (обычно 9:00-21:00)

**Ключевые особенности предметной области**:
- Простая структура данных (3 основные сущности)

4. Система уведомлений для лучшей коммуникации с клиентами
